# OpenFish v1.2.0

OpenFish v1.2.0 focuses on long-running autonomous execution. The headline addition is **Autopilot**, a supervisor-worker mode that keeps pushing a task forward without requiring the human to repeatedly type “continue”.

## Highlights

- Autopilot long-task mode with a dedicated supervisor session (`A`) and worker session (`B`)
- Telegram commands for creating, observing, pausing, resuming, stopping, stepping, and taking over Autopilot runs
- Recent-run management from Telegram panels and buttons
- Stronger runtime observability:
  - run-loop start
  - cycle start
  - worker / supervisor process start
  - current actor
  - current PID
  - elapsed runtime

## Included improvements

### Autopilot runtime

- Added `autopilot_runs` and `autopilot_events` for durable long-task orchestration state.
- Added background Autopilot looping with explicit stop conditions:
  - `complete`
  - `blocked`
  - `needs_human`
  - max cycles
  - repeated no-progress
  - repeated same-instruction loops
- Added manual takeover so a user can inject a higher-level instruction and let Autopilot continue autonomously.
- Added single-step execution while paused for controlled inspection.

### Telegram control surface

- Added `/autopilots` to browse recent runs within the active project.
- Added richer Autopilot status and context cards with:
  - verdict
  - concerns
  - next step
  - recent event timeline
  - latest worker blockers and recommended next step
  - latest supervisor reason and next instruction for the worker
- Added direct Telegram button actions for:
  - status
  - context
  - pause
  - resume
  - stop
  - step
  - takeover

### Diagnostics

- Added runtime instrumentation so a user can distinguish:
  - run created but loop not started
  - loop started but first cycle not entered
  - cycle started but worker not launched
  - worker launched and still running
- Added clearer startup-state messaging for fresh Autopilot runs before the first worker result is available.

## Release quality

- Automated test suite passing (`336 passed`)
- Packaging/build validation remains intact

## Install

```bash
pip install openfish
```

## Upgrade note

`v1.2.0` is additive. Existing `/ask`, `/do`, session, dashboard, and CLI workflows remain intact. Autopilot is an explicit mode for long-running tasks, not a replacement for standard one-shot interactions.
