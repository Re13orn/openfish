# OpenFish v1.0.0

First stable major release of OpenFish.

OpenFish is a single-user, Telegram-driven local Codex assistant for controlling local repositories from Telegram while execution, approvals, state, and audit data remain on the local machine.

## What v1.0 means

This release establishes the stable `v1.x` product surface for:

- single-user scope
- Telegram-first control model
- local-first execution
- stable core command semantics
- stable installation and startup path

## Stable core surface

OpenFish v1.0.0 includes:

- project lifecycle management
- task lifecycle management
- approval flow
- scheduling
- project memory
- unified Codex session browser and import flow
- MCP inspection and toggle controls
- chat-scoped model selection
- self-update and service control commands
- Telegram-first panels, wizards, and status cards

## Included capabilities

- `/projects`, `/use`, `/status`
- `/ask`, `/do`, `/resume`, `/retry`, `/cancel`
- `/approve`, `/reject`
- `/memory`, `/note`, `/diff`
- `/sessions`, `/session`, `/session-import`
- `/schedule-*`
- `/mcp`, `/mcp-enable`, `/mcp-disable`
- `/model`
- `/version`, `/update-check`, `/update`
- `/restart`, `/logs`, `/logs-clear`
- upload analysis controls

## Release quality

- Automated test suite passing (`223 passed`)
- RC validation completed
- Manual smoke test completed
- Runtime startup/status path verified

## Scope and non-goals

OpenFish is intended for:

- one trusted owner
- local repositories
- Telegram as the control surface

OpenFish is not intended to be:

- a multi-user bot platform
- a public remote shell
- a cloud orchestration system

## Upgrade note

`v1.0.0` follows the `v1.0.0-rc1` validation track. Future `v1.x` releases should preserve the current command surface and interaction model except for clearly documented additive improvements.

## Since RC1

- Added unified OpenFish/native Codex session browsing and import.
- Added Telegram-native restart, logs, log clearing, version, and self-update commands.
- Made `stream` the default Telegram UI mode for new chats.
- Fixed same-project follow-up context continuity for `/ask`, `/do`, and plain-text messages.
- Hardened Telegram polling by separating update-polling and outbound-delivery connection pools.
