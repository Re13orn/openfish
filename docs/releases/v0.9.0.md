# OpenFish v0.9.0

First public usable release of OpenFish.

OpenFish is a single-user, Telegram-driven local Codex assistant for controlling local repositories from Telegram while execution, approvals, state, and audit data remain on the local machine.

## What’s included

- Telegram-first workflow with home keyboard, panels, and button-first interaction
- Core task loop: `/ask`, `/do`, `/resume`, `/retry`, `/cancel`
- Approval flow with explicit `approval_id`, note/reason support, and safer callbacks
- Project lifecycle: list, switch, add, disable, archive, default root
- Project memory with notes and paginated `/memory`
- Periodic task scheduling and recovery improvements
- MCP inspection and enable/disable management from Telegram
- Chat-scoped model switching with `/model`
- Streaming progress improvements, typing indicators, and better long-result delivery
- Stronger runtime stability: process lock, restart recovery, network retry
- Refactored persistence layer with focused stores

## Install

```bash
bash mvp_scaffold/scripts/install_start.sh install
bash mvp_scaffold/scripts/install_start.sh configure
bash mvp_scaffold/scripts/install_start.sh check
bash mvp_scaffold/scripts/install_start.sh start
```

## Scope

OpenFish is built for one trusted owner operating local projects through Telegram. It is not a multi-user bot platform or a public remote shell.
