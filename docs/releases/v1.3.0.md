# OpenFish v1.3.0

OpenFish v1.3.0 shifts Telegram from a command-heavy remote control into a more assistant-like local workflow. The release focuses on three product changes:

- natural-language-first interaction
- stronger Telegram mobile UX with buttons and guided flows
- a unified live status surface instead of multiple competing live cards

## Highlights

- Natural language can now route common requests without explicit `/xxx` commands:
  - ask / analyze
  - do / execute
  - autopilot
  - note
  - schedule creation
  - project switching
  - GitHub clone
  - GitHub repo import
- Ambiguous requests no longer default to `/do`; OpenFish now asks for clarification and then continues the original request after you choose the intent.
- Telegram live state is now centered on a single home dashboard card, with detail views available on demand.

## Included improvements

### Natural language entrypoint

- Added heuristic intent routing so plain Telegram text no longer always maps to `/ask`.
- Added project disambiguation and auto-resume:
  - if one project is obvious, OpenFish switches automatically
  - if multiple projects match, OpenFish asks you to choose and then continues the original request
- Added natural-language handling for:
  - project switching
  - schedule creation
  - note creation with lightweight category inference
  - GitHub repository cloning
  - bare GitHub repository links as project-import onboarding

### Telegram mobile UX

- Added more inline-button-first flows for daily use:
  - approvals
  - task follow-up actions
  - project switching
  - Autopilot control
- Long task output is now easier to consume on mobile:
  - shorter primary replies
  - explicit "view full output" actions
  - file delivery for oversized output
- Pending one-shot input modes now persist across service restarts, so tapping "提问" / "执行" / "人工接管" no longer loses state if the service restarts.
- Main menu keyboard is now state-aware for running tasks and pending approvals.

### Unified live status

- The live Telegram home dashboard is now the primary always-on status surface.
- Automatic duplicate live cards were reduced:
  - current task live card is no longer a separate always-on stream
  - Autopilot live state is summarized into the home dashboard
- Detailed views remain available when needed:
  - current task
  - Autopilot status/context/log
  - Autopilot raw stream

### Autopilot usability

- Added persistent raw-stream history and `/autopilot-log`.
- Improved run-specific button routing so pause/resume/stop/takeover actions always target the visible run.
- Added bootstrap instructions from project templates to help the worker start correctly in structured workspaces.
- Simplified supervision so the controller focuses on whether the worker is hesitating or truly blocked.

## Release quality

- Automated test suite passing (`398 passed`)
- Existing CLI, dashboard, and Autopilot workflows remain compatible

## Install

```bash
pip install --upgrade openfish
```

## Upgrade note

`v1.3.0` is additive. Power-user commands still exist, but the intended Telegram path is increasingly:

1. say what you want in plain language
2. let OpenFish route it
3. inspect results and tap the next action
