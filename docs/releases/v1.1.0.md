# OpenFish v1.1.0

OpenFish v1.1.0 focuses on productization after the `v1.0` baseline: PyPI installation, `openfish` CLI as the main entrypoint, richer Telegram dashboards, and a more isolated Docker operating model.

## Highlights

- `pip install openfish`
- `openfish` CLI is now the primary way to install, configure, check, start, stop, restart, inspect logs, and check updates
- Telegram home dashboard via `/home`
- Telegram service dashboard via `/health`
- `OPENFISH_HOME` home-runtime mode with packaged runtime resources
- Docker-specific setup, health, and Codex auth flows

## Included improvements

### CLI and packaging

- Added native `openfish` commands for:
  - `install`
  - `configure`
  - `check`
  - `install-start`
  - `start`
  - `stop`
  - `restart`
  - `status`
  - `logs`
  - `logs-clear`
  - `version`
  - `update-check`
  - `update`
  - `uninstall`
  - `init-home`
- Bundled runtime schema and migrations inside the Python package.
- Published OpenFish on PyPI.

### Telegram UX

- Added a home dashboard that aggregates:
  - service state
  - current project
  - current task
  - current model
  - current session
  - approval state
  - schedule state
- Added a service dashboard with:
  - health
  - version
  - update check
  - update
  - restart
  - logs
  - log clearing
- Added `/ui-reset` to reset a chat back to the default UI mode.

### Docker

- Added Docker-specific setup and health commands.
- Added Docker-side Codex auth import and status helpers.
- Moved Docker runtime toward a dedicated, isolated home/workspace model.
- Hardened Docker startup validation around invalid Telegram token configuration.

## Release quality

- `twine check` passed
- Clean wheel installation verified in a fresh environment

## Install

```bash
pip install openfish
```

## Upgrade note

If you already run OpenFish from a git checkout, the existing repo-based flow remains valid. `v1.1.0` mainly strengthens package-first and home-runtime operation.
