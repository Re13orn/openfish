# OpenFish v1.0.0-rc1

Release candidate for the first stable major version of OpenFish.

This RC is intended to freeze the core product surface:

- single-user scope
- Telegram-driven control model
- local-first execution
- stable core command semantics
- stable install/start path

## Scope of this release candidate

OpenFish v1.0.0-rc1 includes the current core surface:

- project lifecycle management
- task lifecycle management
- approval flow
- scheduling
- project memory
- MCP inspection and toggle controls
- chat-scoped model switching
- Telegram-first panels, wizards, and status cards

## What changed for RC readiness

- Added `v1.0` release readiness checklist
- Added manual smoke test checklist
- Cleaned release documentation flow
- Fixed runtime path resolution for `.env` relative paths
- Improved background startup reliability in `install_start.sh`

## Current release intent

This build should be treated as:

- feature-frozen for the current major version surface
- open for bug fixes, polish, and documentation adjustments only
- the final validation step before `v1.0.0`

## Validation status

- Automated tests passing locally
- Manual feature validation reported complete for the current command surface
- Runtime startup/status regression fixed before RC preparation

## Next step after RC

If no blocking issues are found during RC validation, the next release should be:

- `v1.0.0`
