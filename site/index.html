What OpenFish is for

Built for one trusted owner who wants to keep execution local while still managing AI work from a phone.

OpenFish is designed for

  • Single-user operation
  • Project-scoped continuity
  • Conservative local execution
  • Mobile-friendly Telegram control

OpenFish is not

  • A multi-user bot platform
  • A public remote shell
  • A cloud orchestration layer

Core capabilities

A focused control layer for local Codex workflows instead of a generic remote shell.

Natural-language Telegram entry

Plain text routes to ask, do, autopilot, note, schedule, digest, or project switch without relying on strict slash commands.

Project and task lifecycle

Create, switch, and manage project contexts quickly. Templates can bootstrap new work and trigger autopilot runs.

Local-first privacy

Code, logs, approvals, and runtime state stay on your machine so the owner keeps direct control of the environment.

Autopilot mode

Supervisor-worker execution with pause, resume, stop, raw-output review, and takeover actions from Telegram.

Scheduling and cron

Turn chat instructions like "every 30 minutes check service health" into scheduled automation directly from Telegram.

Docker isolated runtime

Use `openfish docker-init` to provision an isolated runtime for AI execution when you want tighter environment boundaries.

How autopilot works

OpenFish uses a supervisor-worker loop with explicit human controls, instead of hiding long-running execution behind a black box.

Human (Telegram)
Supervisor
Evaluates status and issues the next instruction.
Loop
Worker
Executes tasks, returns output, and waits for the next step.
Human controls: /pause, /resume, /stop, /takeover

Install and bootstrap

Requires Python 3.12+. From install to first Telegram control panel in about a minute.

bash
# 1. Install from PyPI
pip install openfish

# 2. Bootstrap and start
openfish install
openfish configure
openfish check
openfish start

> System ready. Open Telegram and send /home.
